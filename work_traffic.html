Traffic-AI: Predictive Traffic Forecasting

Deep Learning Cloud Computing Python Graph Neural Networks

Project Overview

Traffic-AI is a cloud-native predictive platform designed to transform urban mobility. By fusing high-velocity live traffic data with external feeds like weather conditions and accident reports, the system provides proactive congestion risk assessments and optimal departure times. As the lead for model development, I engineered the core deep learning pipelines that power the system's predictive capabilities.

Data Pipeline & Preparation

Handling city-scale traffic data required a robust preprocessing strategy to ensure model accuracy and stability:

  • Data Fusion: Integrated real-time speed bands from LTA DataMall with weather API data (rainfall, visibility) and incident reports.
  • Cleansing & Imputation: Developed automated scripts to handle missing sensor values and remove outliers caused by GPS drift or sensor malfunction.
  • Feature Engineering: Created spatial-temporal features, including historical congestion windows, holiday flags, and road adjacency matrices for graph-based modeling.
  • Normalization: Applied Min-Max scaling to traffic speeds and standard scaling to weather metrics to ensure stable gradient descent during training.

Model Development

The system utilizes two distinct predictive engines to address different user needs: immediate point-to-point routing and long-term city-wide forecasting.

Short-Term Route Prediction

This module predicts specific travel times between two points by modeling the complex spatial dependencies between road segments.

Key Architectures Explored:

  • GraphWaveNet: Leveraged adaptive dependency matrices to capture hidden spatial relationships between roads without a predefined graph structure.
  • Custom STGNN (Spatial-Temporal Graph Neural Network): Designed a custom architecture combining Graph Convolutional Layers (for spatial features) with Gated Linear Units (for temporal features).
  • Pretrained STGCN: Fine-tuned a Spatial-Temporal Graph Convolutional Network to benchmark performance against custom solutions.

Outcome: The graph-based approach successfully captured the "ripple effect" of congestion, where a jam on one road propagates to connected segments, significantly outperforming traditional statistical baselines.

City-Scale 24-Hour Forecasting

This module generates a full-day congestion outlook for the entire city, enabling proactive planning for commuters and authorities.

The Winning Model: Hybrid LSTM-XGBoost

We evaluated multiple state-of-the-art architectures including Informer, Temporal Fusion Transformer, and N-BEATS. The hybrid approach yielded the best balance of accuracy and computational efficiency.

  • Architecture: An LSTM network captures sequential time-series dependencies (morning/evening peaks), while XGBoost handles tabular features (weather, accidents, holidays).
  • Performance: Achieved an R² of 0.742 and a Mean Absolute Error (MAE) of 4.68 km/h.
  • Efficiency: Demonstrated superior scalability and fast CPU inference times compared to pure Transformer-based models.
