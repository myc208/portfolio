HDB Resale Market & Community Analytics

Hybrid Architecture MySQL MongoDB Data Analytics

Project Overview

Navigating the Singapore housing market can be daunting. This project delivers a unified platform that combines rigorous market data analysis with community-driven insights. By leveraging a Hybrid Database Architecture, the system provides reliable transaction history for buyers while fostering active community discussions.

Data Preparation & Cleansing

The foundation of accurate analytics lies in clean data. I implemented a comprehensive ETL (Extract, Transform, Load) pipeline using Python to process raw HDB resale datasets.

  • Data Cleaning: Standardized flat types (e.g., merging "3 ROOM" and "3-ROOM") and normalized town names to ensure consistency across decades of records.
  • Geospatial Enrichment: Integrated MRT station coordinates to calculate "Proximity Scores," adding value to property listings by quantifying convenience.
  • Anomaly Detection: Identified and flagged outliers in price-per-square-foot metrics to prevent skewing the trend analysis algorithms.

Hybrid Database Architecture

The core technical challenge was selecting the right database for distinct data types. We moved beyond a "one-size-fits-all" approach by integrating Relational (SQL) and Non-Relational (NoSQL) systems.

Handling Structured Transaction Data

Why SQL? The HDB resale records are highly structured, requiring strict schema enforcement and complex join operations to link flats, towns, and amenities.

  • ACID Compliance: Ensures that financial data and transaction records remain consistent and reliable, which is critical for pricing analytics.
  • Complex Queries: Enabled the development of robust analytics APIs, such as /api/analytics/price-by-town (295ms response) and /api/analytics/price-trends, allowing users to filter millions of records efficiently.
  • Normalization: Reduced data redundancy by normalizing town and street data into separate tables.

Powering Community Engagement

Why MongoDB? The social features, such as user comments and "likes," require high write throughput and a flexible schema to accommodate evolving user-generated content.

  • High Scalability: MongoDB's document-oriented structure allows for rapid retrieval of nested comments without expensive JOIN operations.
  • Atomic Operations: Implemented the $inc operator for the "Like" feature. This ensures that even with concurrent user actions, the like counts remain accurate without race conditions.
  • Flexibility: The schema-less design allowed us to easily add new features (like user badges or image attachments) to comments without downtime or complex migrations.
