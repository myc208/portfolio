Enterprise Network Design

A group project designing a scalable enterprise network within `103.30.215.0/24` using Cisco Packet Tracer, featuring VLAN segmentation and a special access room.

Project Description

This group assignment involved designing an enterprise network using the range `103.30.215.0/24`. Configurations made within this project included subnetting the range into /26 segments, configuring VLANs for a Network Lab and a Network Security Lab in block E2, which houses 41 PCs each, while still accomodating to personal devices connected through a Wi-Fi accesspoint. On top of that, the building also encompasses an administration office which has 4 computers and 1 network-enabled printer, which should only be accessible from the administration office itself. In block E6, it consists of 2 Labs with 25 and 21 computers respectively with 1 PC in each of the rooms dedicated to the lecturer. Additionally, there will also be 2 servers in the same block within a special Server Room with unique trunking, and setting up OSPF routing. One of the servers provides DNS for domain name resolution services and the other one represents a web server providing a simple website ("www.singaporetech.edu.sg") for students. However, every device within the same building must be able to ping any other devices within the same building. The design was validated with a Cisco Packet Tracer file and a report justifying our approach.

Key Features (My Contribution)

VLAN Segmentation

Subdivided `103.30.215.0/24` using CIDR into /25 for each of the 2 Network Labs in E2, /29 for the Adminstration Office, /26 for the Labs in E6, and /30 for the Server Room.

Inter-VLAN Routing

Implemented OSPF to route between VLANs within the constrained IP range.

Special Room Access

Configured VLAN 30 with trunking for restricted Server Room access.
