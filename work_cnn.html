Development of Convolutional Neural Network

Comparing self-trained models to pre-trained model (VGG16)

Project Description

Develop a neural network model for prediction accuracy comparison with other pre-trained learning algorithms such as VGG16, ResNet, InceptionV3, and MobileNet. The outcome of the project is to identify an algorithm that most accurately classifies an image to the name of the food that is shown within.

Technologies Used

  • Google Collab
  • Python
  • Search Engine
  • Provided dataset

Summary of Steps

  • Develop a self-trained model
  • Research on various pre-trained models
  • Develop model based on selected choice of pre-trained models
  • Analyse results
  • Use untrained data for testing
  • Record analysis results and findings

Choice of pre-trained model

VGG16 was chosen due to the simplicity to understand it and few of the advantages that it provides in comparison to other pre-trained models. These benefits include:

  • Image classification benefits greatly from transfer learning
  • Dataset was small and not complex
  • Simple model structure aids educational learning
  • Such a simple model structure sets a good baseline score for pre-trained models
  • Consists of deep convolutional layers to capture rich features essential for image classification

Results

Observations

  1. Model performance:

    The VGG16 pre-trained model performs better in terms of accuracy compared to the self-trained CNN model. This is likely due to the extensive feature extraction capabilities of VGG16, which has been trained on a much larger dataset, enabling it to recognize more complex patterns.

  2. Training Efficiency:

    Despite the smaller batch size and increased evaluation steps, the pre-trained model utilizing VGG16 still achieved better accuracy, suggesting that it handles smaller, more frequent updates effectively, which might have contributed to its superior performance.

  3. Model Architecture:

    The pre-trained VGG16 model has a much larger parameter count and leverages pre-trained weights. This makes it more powerful in recognizing complex features, whereas the self-trained CNN model, with fewer parameters and no pre-trained knowledge, performs less effectively in comparison.

  4. Inference Time:

    The inference time per step for the VGG16 model is slightly shorter (74ms/step) than the self-trained model (106ms/step), despite the higher parameter count. This might be due to VGG16's optimized architecture for image recognition tasks.

  5. Dense layer design impact:

    The additional dense layer in VGG16 allows for more abstracted feature transformation before the final classification, which could contribute to its better accuracy, as it enables a more refined mapping between extracted features and classes.
