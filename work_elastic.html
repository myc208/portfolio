Establishing ElasticSearch DB Ecosystem

Test viability, efficiency and effectiveness of ElasticSearch DB over MSSQL for speed and functionality.

Project Description

As a database engineer intern, my task was to establish a working database on elasticsearch and learn about the ecosystem that revolves around it, known as the elasticstack. Through that, I can simplify the implementation process of various other features such as real-time data logging, data visualizations, as well as the community discussion group to query and seek professional help from.

Technologies Used

  • Google Meet
  • Python
  • Google collab
  • MSSQL
  • Docker
  • Kibana
  • Logstash
  • ElasticSearch
  • Provided sample dataset

Steps for Primary Task

Establishing DB and ensure connectivity within ElasticStack ecosystem

  • Research and learn about ElasticSearch
  • Implementation of ElasticSearch DB on Docker (containerization platform)
  • Check DB status on Kibana (visualization tool)
  • Attempt connection to MSSQL using Logstash (data processing pipeline)
  • Implement real-time data logging feature
  • Log and record obstacles faced and solutions implemented
  • Test full functionality with sample data in MSSQL

Steps for Secondary Task

Develop flagging algorithm for logged data

  • Implement sample words to be flagged
  • Develop flagging algorithm
  • Test algorithm functionality
  • Log and record obstacles faced and solutions implemented
  • Send algorithm to supervisors for UAT
  • Make changes based on feedbacks before repeating development and testing process

Working in a team setting

  • Perform daily standup meetings in the morning to check in on each individual's goals for the day
  • Many seniors to consult when faced with impediments
  • Lots of learning opportunities from others in the same field
  • Increased engagement which elevates work experience
